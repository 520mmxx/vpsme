# 文件权限说明

OpenDeepSeek 的核心价值是“真 Agent”：它不仅聊天，还能读取文件、生成网页、写报告、保存脚本、整理资料。要做到这些，Hermes 需要访问你授权给它的本机目录。

## 默认推荐目录

中国版默认建议使用专用目录：

```text
~/OpenDeepSeek-Agent/
  OpenDeepSeek-Inputs/
  OpenDeepSeek-Outputs/
  OpenDeepSeek-Memory/
```

含义：

- `OpenDeepSeek-Inputs`：你主动放进去让 Agent 读取的文件
- `OpenDeepSeek-Outputs`：Agent 生成的网页、报告、脚本、周报
- `OpenDeepSeek-Memory`：长期记忆和偏好文件

这样做的好处是安全边界清楚，适合小白试用、录视频和分享给朋友。

## 如果要访问桌面、下载、文档

需要扩大 `.env` 里的：

```env
HERMES_HOST_DIR=/Users/你的用户名
```

或指定某个目录：

```env
HERMES_HOST_DIR=/Users/你的用户名/Desktop
```

重启后，Hermes 在容器内通过 `/host` 访问这个目录。

例子：

```text
HERMES_HOST_DIR=/Users/lauralyu
```

用户在聊天里可以说：

```text
请查看 /host/Desktop 有什么，但不要移动或删除文件，只生成整理建议。
```

## 重要安全规则

- 不要在无登录状态下把 OpenDeepSeek 暴露到公网。
- 不要把 `HERMES_HOST_DIR` 指向你不想让 Agent 看到的目录。
- 首次使用建议只让 Agent 读文件名，不读取正文。
- 涉及删除、移动、覆盖文件时，应要求 Agent 先列计划，确认后再执行。
- `.env` 里不要写任何要提交到 Git 的秘密。

## 生成的文件在哪

建议统一要求 Agent 保存到：

```text
/host/OpenDeepSeek-Outputs/
```

如果 `HERMES_HOST_DIR=~/OpenDeepSeek-Agent`，本机路径就是：

```text
~/OpenDeepSeek-Agent/OpenDeepSeek-Outputs/
```

下一阶段的 Artifact Manifest 会把这些产物显示成卡片，用户可以直接预览和打开。

## 常用提示词

```text
请在 /host/OpenDeepSeek-Outputs/site 里生成一个单文件 index.html。
```

```text
请读取 /host/OpenDeepSeek-Inputs 里的资料，生成总结到 /host/OpenDeepSeek-Outputs/summary.md。
```

```text
请查看 /host/Desktop 的文件名，不读取正文，不移动文件，先给我整理方案。
```
