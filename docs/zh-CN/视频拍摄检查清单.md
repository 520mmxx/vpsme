# OpenDeepSeek 视频拍摄检查清单

## 录前 5 分钟

先开 Docker/OrbStack，再在项目目录运行：

```bash
./setup.sh start
./scripts/creator-demo.sh
```

8 项全绿再开录：

1. GitHub main 已同步当前版本。
2. Gitee 镜像可访问。
3. Open WebUI / Smart Bridge / Hermes 容器健康。
4. DeepSeek API Key 真能返回内容。
5. 普通问答走轻量快路径。
6. `/agent` 能真实写入文件。
7. Cron/提醒任务能真实创建。
8. Portal `http://localhost:3001` 可打开。

如果要演示联网搜索，再单独启动：

```bash
./setup.sh start-full
```

低内存电脑不要默认开 full。

## 录视频时关掉的应用

- Codex.app 或 Claude Code 里正在跑的长任务。
- Activity Monitor / 任务管理器。
- WeChat、微信、Slack、飞书、钉钉等会弹窗的 IM。
- 其他本地开发服务器，尤其是高 CPU 的 Node/Python 进程。
- 不需要的浏览器标签页。

建议用无痕窗口录制，浏览器缩放 110% 到 125%，字体要让手机观众也看得清。

## 5 步录制脚本

1. 打开 `http://localhost:3001`，一句话介绍三件套：

   > Open WebUI 做界面，Hermes 做真 Agent，DeepSeek V4 Flash 做默认低成本推理。

2. 填 DeepSeek Key，选择默认 DeepSeek，点激活，跳到 `http://localhost:3000`。

3. 普通问答测试：

   ```text
   你好，介绍下你自己。用三句话说明普通问答和真 Agent 有什么区别。
   ```

   重点展示：回复快、中文、不是直接进长任务。

4. 真 Agent 文件测试：

   ```text
   /agent 请在 /host/OpenDeepSeek-Outputs/hello.txt 写入“你好世界”，完成后验证文件存在且大小大于 0。
   ```

   重点展示：不是口头答应，Finder 里能看到真实文件。

5. Cron/提醒测试：

   ```text
   /agent 请创建一个 30 分钟后提醒我喝水的任务，并告诉我任务 ID。
   ```

   重点展示：Hermes 负责真实后台任务。

## 常见翻车防范

- 录制前跑 `./scripts/creator-demo.sh`，不要跳过。
- 演示前清空或新建聊天，避免旧上下文影响路由。
- 不要同时打开 Codex、Claude、OpenClaw 长任务，电脑会抢资源。
- 需要搜索才开 `./setup.sh start-full`，普通演示只用 `./setup.sh start`。
- 如果页面卡住，先跑 `./setup.sh doctor`，看是不是 Docker daemon 没开或旧 `.env` 还在 full 模式。
- 如果 Agent 说“已生成”但找不到文件，立刻停止录制，跑 `./scripts/creator-demo.sh` 的第 6 项排查。
- 如果 Open WebUI 记住了旧配置，到 Admin 设置里确认默认模型是 `opendeepseek-auto`。

## 录完释放资源

```bash
./setup.sh stop
```

这只停止容器，不删除聊天记录和 volume。
